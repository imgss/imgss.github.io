如何将hexo博客部署到github子路径上

hexo有一个脚手架工具，然后还有一个hexo d命令，在_config.yml文档中配置一下服务器的路径，可以自动部署到github上，于是，我就试了一下这个命令。然后，我的主页就被替换了。。。就是整个.github.io仓库完全被替换了，所以我应该说一下自己爬过的坑。

配置_config.yml

hexo是将md文档转换成一系列漂亮的html文档的工具，而且可以换漂亮的皮肤主题，得到一个漂亮的静态博客网站。

那么，hexo是怎么知道html页面引用的css，js的路径关系呢，答案就是这个配置文档了，如果配置不正确，此时css和图片的加载就不会正确进行，然后就会得到一个上世纪90年代的页面。正确的引用配置在url字段中。

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: https://imgss.github.io
root: /blog/

上面是我的配置，就是将博客的根路径，放在https://imgss.github.io/blog下面。配置好之后，使用hexo g命令就可以从source目录的md文件正确的在public文件夹下生成整个博客的所需的所有资源文件，主要包括css,js，和archives(文章目录)。

部署

前面已经说过，直接使用hexo d命令会将博客部署在站点的根路径下面。所以，不能直接使用这个命令。那么该怎么办呢，就是在你本地的xxx.github.io仓库文件夹下面，新建一个blog文件夹，然后把刚才public文件夹下面的所有文件复制到blog文件夹下，然后push到远程仓库，然后你就在子路径下部署了这个博客。
最后在你的主页上加一个导向这个路径的超链接，就大功告成了。
是不是很简单？当然，这么做几次，你就会发现一旦你写了文章，就要一直去把新生成的文件复制到blog文件夹下，这时就显得很蠢了。。

最后

突发奇想，可以写一个自动移动到目标文件夹下的命令也是极好的。可以努力一下。可以用node，可以用python的shutil，但还是批处理最简单：

xcopy e:\tmp\blog\public e:\github\imgss.github.io\blog /e/h

把上面的命令的两个e开头的路径换成你hexo的生成博客的public路径，另一个是你github博客的部署路径，然后保存成一个记事本里

后缀名改为.bat，每次更新博客到public里了，点这个文件一下，就ok。
